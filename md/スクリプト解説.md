作成日：2026/05/07

## 全体の流れ

スクリプトは「段階的に銘柄を絞り込む」設計になっています。
上から順番に条件を確認し、1つでも引っかかった時点でその銘柄は除外されます。
「全部チェックしてから判定する」のではなく「ダメなものを早めに弾く」方式です。

```
CSVを読み込む
  ↓
STEP1：財務フィルター（CSVのデータだけで判定）
  ↓
STEP2：当日騰落フィルター（その日の値動きで判定）
  ↓
Q1：日経平均の地合い確認（ネットからリアルタイムデータ取得）
  ↓
Q4〜Q11：個別銘柄のチャート分析（ネットからリアルタイムデータ取得）
  ↓
結果をコンソールに表示 ＋ resultsフォルダにテキスト保存
```

## STEP1：財務フィルター

**使うデータ：楽天証券のCSVファイル（ネット接続不要）**

CSVに含まれる数字だけを使って判定します。ここは高速で動きます。

### チェック内容

| 項目 | 基準 | 理由 |
|------|------|------|
| 売上高変化率（前年度比） | +20%以上 | 成長株の最低ライン。売上が伸びていない会社は対象外 |
| 経常利益変化率（前年度比） | +20%以上 | 利益が増えていない会社も対象外。EPS成長の代替指標として使用 |
| 自己資本比率 | 50%以上 | 財務の安全性。50%未満は借金が多すぎる可能性がある |
| 時価総額 | 100億〜1,500億円 | 小さすぎると流動性リスク、大きすぎると値動きが鈍い |

### スクリプト内での変数名
```python
MIN_REVENUE_GROWTH = 20.0   # 売上高成長率の下限
MIN_PROFIT_GROWTH  = 20.0   # 利益成長率の下限
MIN_EQUITY_RATIO   = 50.0   # 自己資本比率の下限
MIN_MARKET_CAP     = 10000  # 時価総額の下限（百万円）= 100億円
MAX_MARKET_CAP     = 150000 # 時価総額の上限（百万円）= 1,500億円
```

## STEP2：当日騰落フィルター

**使うデータ：CSVの「前日比(%)」列**

その日に大きく動きすぎた銘柄を除外します。

### チェック内容

- **当日の騰落率が±5%を超えていたら除外**

### なぜこれが必要か
- +5%以上：すでに大きく上がってしまっている。今から買うと「高値掴み」になりやすい
- -5%以下：何か悪いことが起きている可能性がある。原因不明のまま買うのは危険

### スクリプト内での変数名
```python
MAX_DAY_CHANGE_PCT = 5.0  # ±5%超で除外
```

## Q1：日経平均の地合い確認

**使うデータ：Yahoo Finance からリアルタイムで取得（ティッカー：^N225）**

個別銘柄を調べる前に、市場全体の状態を確認します。
川が逆流しているときに泳いでも無駄なのと同じで、相場全体が下向きのときに個別株を買っても勝てません。

### チェック内容

1. **日経平均の終値が25日移動平均線より上か**
2. **25日移動平均線が上向きか**（10日前の25日線と比較）

### 判定結果

| 状態 | 判定 | 次のアクション |
|------|------|----------------|
| 25日線の上 × 上向き | 強気 → 通過 | チャート審査に進む |
| 25日線の上 × 横ばい | 中立 → 通過 | チャート審査に進む（慎重に） |
| 25日線の下 | 弱気 → 全面見送り | ここで分析終了 |

### スクリプト内の処理
```python
# 3ヶ月分のデータを取得
nikkei = yf.Ticker("^N225")
hist   = nikkei.history(period="3mo")

# 25日移動平均を計算
hist['MA25'] = hist['Close'].rolling(25).mean()

# 最新値と10日前を比較して「上向きか」を判定
ma25_now = hist['MA25'].dropna().iloc[-1]   # 今日の25日線
ma25_old = hist['MA25'].dropna().iloc[-10]  # 10日前の25日線
ma25_rising = ma25_now > ma25_old           # 上向きかどうか
```

## Q4：移動平均線の配列（パーフェクトオーダー）

**使うデータ：yfinance から14ヶ月分の株価データを取得**

移動平均線とは「過去N日間の終値の平均」です。
パーフェクトオーダーとは、短期線・中期線・長期線がきれいに上から順に並んだ状態で、上昇トレンドが強い証拠です。

### チェック内容

```
現在の株価 > 50日移動平均 > 150日移動平均 > 200日移動平均
```

この順番が崩れていたら即【見送り】。

### なぜこの4本か
- **50日線**：約2ヶ月の平均。中期的なトレンドを表す
- **150日線**：約6ヶ月の平均。中長期的な傾向を表す
- **200日線**：約1年の平均。長期的な大きな流れを表す

### スクリプト内の処理
```python
hist['MA50']  = hist['Close'].rolling(50).mean()
hist['MA150'] = hist['Close'].rolling(150).mean()
hist['MA200'] = hist['Close'].rolling(200).mean()

# 全部が順番通りに並んでいるか
q4_pass = bool(close > ma50 > ma150 > ma200)
```

## Q5：200日移動平均線の方向

**使うデータ：Q4と同じ（追加取得なし）**

200日線がどちらを向いているかを確認します。

### チェック内容

- **現在の200日線 > 30日前の200日線** → 上向き → 通過
- それ以外 → 下向き/横ばい → 見送り

### なぜ大事か
200日線が下向きということは、長期的に株価が下落傾向にあることを意味します。
そのような銘柄をいくら短期で買っても、長期の力に押し戻されやすいです。

### スクリプト内の処理
```python
ma200_30ago = hist['MA200'].dropna().iloc[-31]  # 30日前の200日線
q5_pass = bool(ma200 > ma200_30ago)             # 今の方が高ければ上向き
```

## Q6：出来高の質

**使うデータ：Q4と同じ（追加取得なし）**

出来高（売買された株数）が「上昇日」と「下落日」でどちらが多いかを比べます。

### チェック内容

直近20営業日を調べ、
- **上昇した日の平均出来高** ÷ **下落した日の平均出来高** = 比率

| 比率 | 意味 | 判定 |
|------|------|------|
| 1.3倍以上 | 上がる日に活発に買われている | 通過 |
| 0.8〜1.3倍 | ほぼ横ばい | 通過（条件付き） |
| 0.8倍未満 | 下がる日の方が活発 → 売り圧力が強い | 見送り |

### なぜ大事か
「株価は上がっているが出来高が少ない」場合は、本物の上昇ではなく、売る人が少ないだけの「虚弱な上昇」のことがあります。
逆に「下落日に出来高が多い」場合は、大量の売りが出ていることを意味し危険です。

### スクリプト内の処理
```python
recent = hist.tail(20).copy()
recent['is_up'] = recent['Close'] >= recent['Open']  # その日上がったか

up_vol   = recent.loc[recent['is_up'],  'Volume'].mean()   # 上昇日の平均出来高
down_vol = recent.loc[~recent['is_up'], 'Volume'].mean()   # 下落日の平均出来高
ratio    = up_vol / down_vol  # 比率
```

## Q7：25日移動平均線からの乖離率

**使うデータ：Q4と同じ（追加取得なし）**

現在の株価が25日移動平均線からどれだけ離れているかを計算します。

### チェック内容

```
乖離率(%) = (現在株価 - 25日線) ÷ 25日線 × 100
```

| 乖離率 | 意味 | 判定 |
|--------|------|------|
| +5%以内 | 25日線に近い。健全な状態 | 通過 |
| +5〜15% | やや過熱気味。注意が必要 | 通過（条件付き） |
| +15%超 | 過熱しすぎ。反落リスクが高い | 見送り |
| マイナス（25日線以下） | 上昇勢いが弱い | 通過（慎重に） |

### なぜ大事か
株価は25日線から大きく離れると、ゴムが伸びたように元に戻ろうとする性質（平均回帰）があります。
+15%も上にある銘柄は「今が一番高い時期」に近い可能性があります。

### スクリプト内の処理
```python
dist = ((close - ma25) / ma25) * 100  # 乖離率を計算
```

## Q10：当日のギャップ確認

**使うデータ：Q4と同じ（追加取得なし）**

「ギャップ」とは、前日の終値と当日の始値（寄り付き）の差のことです。

### チェック内容

```
ギャップ(%) = (当日の始値 - 前日の終値) ÷ 前日の終値 × 100
```

| ギャップ | 意味 | 判定 |
|----------|------|------|
| ±5%以内 | 通常の範囲 | 通過 |
| +5%以上 | 大きなギャップアップ | 見送り |
| -5%以下 | 大きなギャップダウン | 見送り |

### なぜ大事か
- **ギャップアップ**：すでに高く始まっている。今から買うと高値掴みになりやすい
- **ギャップダウン**：朝から大量の売りが出ている。何か悪材料が出た可能性がある

### スクリプト内の処理
```python
gap = ((last['Open'] - prev['Close']) / prev['Close']) * 100
# last['Open']  = 今日の始値
# prev['Close'] = 昨日の終値
```

## Q11：直近3ヶ月高値（参考情報）

**使うデータ：Q4と同じ（追加取得なし）**

過去63営業日（≒3ヶ月）の中で、最も高い値（高値）を計算します。
これは「見送り/通過」の判定には使いません。**ブレイクアウト分析のための参考情報**として表示します。

### 使い方
- 現在の株価が3ヶ月高値に近い（97%以上）場合は「高値近辺」と表示
- この価格を超えて上昇する（ブレイクアウト）タイミングが、名探偵モードでのエントリートリガー候補になります

### スクリプト内の処理
```python
high_3m   = hist.tail(63)['High'].max()    # 63日間の最高値
near_high = close >= high_3m * 0.97        # 97%以上なら高値近辺
```

# trade_review.py 解説（成績集計ツール）

`run_analysis.bat` を実行すると `trades/` フォルダに 1銘柄1ファイルの紙トレ記録テンプレートが生成されます。
そこにエントリー価格・決済価格などを手書きで埋めていくと、`review.bat` がその記録を集計して
**Q13 の連敗判定を自動化** してくれます。

## 使い方

1. `run_analysis.bat` で生成された `trades/trade_YYYYMMDD_コード_銘柄名.txt` を開く
2. 実際にエントリーしたら **エントリー価格・株数・損切りライン・目標価格** を記入
3. 決済したら **決済価格・損益・保有日数・決済理由** を記入
4. `review.bat` をダブルクリック

## 出力される内容

| 項目 | 説明 |
|------|------|
| 全体成績 | 勝率・累計損益・平均損益・平均保有日数 |
| 最長連勝/連敗記録 | これまでの最高/最悪の連続記録 |
| ベスト/ワーストトレード | 最大利益・最大損失のトレード |
| **Q13: 連敗チェック** | **直近3連敗なら自動で「1週間停止」を表示** |
| 直近のトレード | 新しい順に5件、勝敗マーク付き |
| 保有中ポジション | まだ決済していない銘柄の一覧 |
| 予定 | テンプレートはあるがエントリーしていない記録 |

## 記録ファイルの記入例

`エントリー価格`・`決済価格` の数値を見て、空欄か0なら未記入と判定します。
カンマ区切り（1,500）でも普通の数字（1500）でもどちらでも読めます。

```
  エントリー価格  : 1,500 円
  株数            : 100 株
  決済価格        : 1,650 円
  損益            : +15,000 円  （ +10.0 % ）
  保有日数        : 7 日
```

損益と保有日数を空欄にしておいても、エントリー/決済価格・株数から自動計算されます。

## Q13 が発動したら

```
  [STOP] 3連敗中! 本日は新規エントリー見送り推奨。
         再開可能日の目安: 2026-05-14 以降
         冷静になる時間を確保し、紙トレ記録を読み返しましょう。
```

連敗の最終決済日 +7日 を「再開可能日の目安」として表示します。
この期間は紙トレ記録（`trades/` フォルダ）の `決済理由` と `反省・学び` を読み返す時間にあててください。

## 結果の保存場所

実行のたびに `results/review_YYYYMMDD_HHMM.txt` として保存されるので、
過去の集計をいつでも見返せます。

# Q8 ニュース半自動チェック 解説

`swing_analyzer.py` に組み込み済みの機能です。別途実行する必要はありません。
`run_analysis.bat` を叩くと、チャート審査を通過した銘柄に対して自動的に実行されます。

## 何をするか

yfinance の `ticker.news` API を使い、対象銘柄の直近ニュース（最大15件）の見出しを取得して
**危険キーワード**・**ポジティブキーワード**をスキャンします。

## 危険キーワード（検出されたら [WARN] 表示）

```
不祥事 / 粉飾 / 行政処分 / 公募増資 / 第三者割当 / 増資
下方修正 / 業績修正 / 不正 / 訴訟 / 大量退任 / 経営交代
業績悪化 / 赤字転落 / 特別損失 / 業績予想を修正 / 経営陣
```

## ポジティブキーワード（検出されたら [OK] 表示 ＋ 加点参考）

```
上方修正 / 自社株買い / 増配 / 株式分割 / 大型受注
資本提携 / M&A / 黒字転換 / 好業績
```

## コンソールの表示例

```
     Q8 ニュースチェック中... [Q8-WARN] 危険キーワード検出: 増資 → 楽天証券で要確認
     Q8 ニュースチェック中... [Q8-OK] ポジティブ: 上方修正
     Q8 ニュースチェック中... [Q8-OK] 危険キーワードなし
     Q8 ニュースチェック中... [Q8-?] ニュース0件（yfinanceで取得できず） → 楽天証券で手動確認
```

## 重要な注意点

| ポイント | 内容 |
|---------|------|
| **yfinanceのニュースは英語が中心** | 日本語のキーワードにはヒットしないことが多い。0件でも「安全確定」ではない |
| **[WARN] = 即除外ではない** | キーワードが見出しに含まれていた = 確認が必要というシグナル。楽天証券Webで目視してから最終判断する |
| **[?] = 不明** | yfinanceが取得できなかった場合。楽天証券Webで必ず手動確認する |
| **最終判断は必ず目視で** | このツールはあくまで「見落とし防止の補助」。楽天証券のニュースタブを自分の目で確認することを推奨 |

# HTML レポート生成 解説

ブラウザで開くと以下の内容が表示されます。

| セクション | 内容 |
|-----------|------|
| Q1 地合いバナー | 🟢強気 / 🟡中立 / 🔴弱気 を色帯で大きく表示 |
| 日経平均チャート | 6ヶ月の終値ライン + MA25 |
| 通過銘柄サマリー表 | 全通過銘柄をスコア順に一覧（グレード・Q8結果・決算日付き）|

```

棒グラフ = 出来高（上昇日=緑、下落日=赤）
```


インストールされていない状態で実行すると、チャートなしのテキスト版HTMLが生成されます（クラッシュしません）。
